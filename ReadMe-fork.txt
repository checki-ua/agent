Данный форк имеет две цели:
1) добавить к библиотеке возможность запуска в виде автономного веб-сервера с выбираемым рабочим портом
2) зафиксировать версию библиотеки и тем самым обезопасить работу основного приложения при возможных несовместимых обновлениях исходного кода.

Веб-сервер сделан в виде нового отдельно файла index-web.js.
В качестве основы для dеб-сервера взят мини-фреймворк Express (https://expressjs.com/ru/).
Функции модуля index-web.js заключаются в прослушивании назначенного порта и передаче полученных команд параметров собственно в сам агент подписей.
Команды и параметры полностью повторяют оригинальные и описаны в README.md, новые парметры дополняют исходные

Агент подписей agent-web.js представляет из себя слегка переработанный оригинальный agent.js.
Цель доработок - приспособить для работы с веб-сервером в формате, удобном для основного приложения.

Запуск веб-сервера:
node index-web.js [--port <номер порта>]

либо c дефолтным портом
npm run start

По умолчанию используется порт 3001



Формат запросов.
Веб-сервер получает в POST-запросе и отдаёт в агент подписей данные цифровой подписи и подписываемые данные, а именно:

Подписать данные:
POST <cryptServerUrl>:<cryptServerPort>/sign

POST query: 'pass' - пароль от электронной подписи
POST body(json):
    'data' - шифрукмые данные
    'rawKey' - файл ключа электронной подписи (Key-6.dat)
    'rawCert1' - сертификат1 электронной подписи (файл EU-cert1.cer)
    'rawCert2' - сертификат2 электронной подписи (EU-cert2.cer, если есть)

Все параметры в PostBody передаются как BASE-64 кодированная строка.

Расшифровать подписанные данные:
POST <cryptServerUrl>:<cryptServerPort>/decrypt

POST query: 'pass' - пароль от электронной подписи
POST body(json):
    'data' - шифрукмые данные
    'rawKey' - файл ключа электронной подписи (Key-6.dat)
    'rawCert1' - сертификат1 электронной подписи (файл EU-cert1.cer)
    'rawCert2' - сертификат2 электронной подписи (EU-cert2.cer, если есть)
    'rawCa' - файл со списком сервисов шифрования (CACertificates.p7b)

Все параметры в PostBody передаются как BASE-64 кодированная строка.

Ответ сервера приходит также BASE-64 кодированным.
